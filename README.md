## ✨ What is this?

**YT Automation** is a local pipeline that writes, voices, edits, and uploads YouTube Shorts automatically. You type a topic. The system does everything else.

```
You: "dark psychology facts"

→  Script written by Llama 3.3 70B via Groq     (~3 seconds)
→  Voiceover generated by Kokoro TTS             (local, free)
→  Stock footage fetched from Pexels             (free API)
→  Vertical 1080×1920 video assembled            (FFmpeg + Pillow)
→  Uploaded to your YouTube channel              (YouTube Data API)
```

**Zero cost. Zero subscriptions. Runs on any Windows PC.**

## ⚡ Quick Start

### 1. Clone the repo

```bash
git clone https://github.com/mzu-2410z/yt-automation.git
cd yt-automation
```

### 2. Create virtual environment

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
pip install python-dotenv
```

### 4. Download Kokoro TTS model files

Run these from inside the project folder:

```bash
curl -L -o kokoro-v1.0.onnx https://github.com/thewh1teagle/kokoro-onnx/releases/download/model-files-v1.0/kokoro-v1.0.onnx
curl -L -o voices-v1.0.bin https://github.com/thewh1teagle/kokoro-onnx/releases/download/model-files-v1.0/voices-v1.0.bin
```

### 5. Create your `.env` file

```bash
cp .env.example .env
```

Open `.env` and fill in your free API keys:

```env
GROQ_API_KEY=your_groq_key_here
PEXELS_API_KEY=your_pexels_key_here
```

Get them free at:
- **Groq** → [console.groq.com](https://console.groq.com) (no credit card)
- **Pexels** → [pexels.com/api](https://www.pexels.com/api/)

### 6. Add YouTube credentials

Follow the [YouTube API Setup Guide](#-youtube-api-setup) below and place `client_secrets.json` in the project root.

### 7. Run

```bash
python main.py
```

First run opens a browser for YouTube login. After that — fully automatic.

## ⚙️ Configuration

Everything is controlled from `config.py`. No need to touch any other file.

### LLM Provider

```python
LLM_PROVIDER = "groq"       # "groq" | "gemini" | "ollama"
GROQ_MODEL   = "llama-3.3-70b-versatile"
```

| Provider | Quality | Speed | Cost |
|---|---|---|---|
| Groq (recommended) | ⭐⭐⭐⭐⭐ | Instant | Free |
| Google Gemini | ⭐⭐⭐⭐⭐ | Fast | Free |
| Ollama (local) | ⭐⭐⭐ | Slow on CPU | Free |

### Voice

```python
KOKORO_VOICE = "af_heart"   # af_heart · af_bella · am_adam · bf_emma · bm_george
KOKORO_SPEED = 0.9           # 0.8 slow · 1.0 normal · 1.2 fast
```

### Video format

```python
OUTPUT_WIDTH  = 1080         # Vertical Shorts format (9:16)
OUTPUT_HEIGHT = 1920
```

### Upload privacy

```python
YOUTUBE_PRIVACY = "private"  # "private" | "unlisted" | "public"
```

> Start with `"private"` to review before publishing. Switch to `"public"` when happy.

### Background music

Drop any royalty-free MP3 into the `music/` folder as `bg.mp3`. The system mixes it in automatically at low volume. Free music at [YouTube Audio Library](https://studio.youtube.com/channel/music).

## 🔑 YouTube API Setup

<details>
<summary>Click to expand — step by step</summary>

1. Go to [console.cloud.google.com](https://console.cloud.google.com/)
2. Click **Select a project** → **New Project** → name it anything → **Create**
3. Search for **YouTube Data API v3** → click it → **Enable**
4. Go to **Credentials** → **Create Credentials** → **OAuth 2.0 Client ID**
5. If asked to configure consent screen:
   - Choose **External** → **Create**
   - Fill in App name and email fields → **Save and Continue** through all steps
6. Back at Credentials → **Create Credentials** → **OAuth 2.0 Client ID**
7. Application type: **Desktop app** → **Create**
8. Click **Download JSON** → rename to `client_secrets.json`
9. Place `client_secrets.json` in the project root folder

First run will open a browser asking you to log in. After that it's fully automatic — your token is saved in `youtube_token.pkl`.

</details>

## 🗓️ Automated Scheduling

To publish daily without touching anything, use Windows Task Scheduler:

1. Search **Task Scheduler** in Start Menu
2. **Create Basic Task** → name it `YT Auto`
3. Trigger: **Daily** → set your time
4. Action: **Start a program**
   - Program: `D:\path\to\yt-automation\venv\Scripts\python.exe`
   - Arguments: `main.py "your topic here"`
   - Start in: `D:\path\to\yt-automation`
5. Finish

For hands-off content, create `topics.txt` with one topic per line and modify `main.py` to pick a random line each run.

## 🛠️ Troubleshooting

| Error | Fix |
|---|---|
| `ffmpeg is not recognized` | Add `C:\ffmpeg\bin` to Windows PATH, restart PC |
| `FileNotFoundError: voices-v1.0.bin` | Run the curl commands in Quick Start Step 4 from inside the project folder |
| `Cannot connect to Ollama` | Run `ollama serve` in a terminal first |
| `client_secrets.json not found` | Complete YouTube API Setup and place file in project root |
| `getaddrinfo failed` on upload | Internet dropped during upload — run again |
| Script too short (under 30s) | Run again — LLM output is non-deterministic, next run will be longer |
| `insufficientPermissions` on upload | Delete `youtube_token.pkl` and re-authenticate |
| Footage overlay falling back to slides | Make sure you have the latest `visual_builder.py` |
